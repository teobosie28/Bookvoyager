DOCUMENTAȚIE PROIECT: BOOKVOYAGER
==================================================
SUMAR EXECUTIV
==================================================

BookVoyager este o aplicație web full-stack construită în Python cu Django Framework, demonstrând principii de programare pe partea de server și gestionarea securizată a datelor. Aplicația permite utilizatorilor să gestioneze recenzii, favorite și să efectueze interogări complexe pe catalogul de cărți.

--------------------------------------------------
ARHITECTURA TEHNOLOGICĂ
--------------------------------------------------

- Limbaj / Framework: Python 3.13 / Django 5.2.1
- Bază de Date: SQLite (Accesată exclusiv prin Django ORM)
- Arhitectură: Model-View-Template (MVT)

==================================================
I. UTILITĂȚI PRINCIPALE ȘI ROLUL PYTHON
==================================================

1. GESTIUNEA DATELOR (ORM)
   - ORM (Object-Relational Mapping): Permite definirea structurii bazei de date (Modelele) prin clase Python. 
   - Conversie Automată: Python traduce automat comenzile de interogare (Ex: Book.objects.filter()) în SQL optimizat.
   - Beneficiu Securitate: Protecție automată împotriva atacurilor de tip SQL Injection.

2. LOGICA DE BUSINESS ȘI INTEROGARE
   - Logica în Views: Toate regulile aplicației (logica de business) sunt executate în funcțiile Python (Views) pe server.
   - Interogare Complexă: Utilizarea QuerySet API (obiecte Q()) pentru a filtra datele pe baza unor condiții multiple (Ex: Căutare după Titlu SAU Autor).

3. SECURITATE ȘI AUTENTIFICARE
   - Autentificare Server-Side: Sistemul folosește Hashing (criptare unidirecțională) pentru verificarea parolelor pe server.
   - Validare Server-Side: Datele din formulare sunt validate de Python pe server înainte de a fi salvate.

==================================================
II. STRUCTURA APLICAȚIEI (MVT)
==================================================

- Modele (models.py): Definirea structurii datelor și a relațiilor (clase Python).
- View-uri (views.py): Executarea logicii de business și interacțiunea cu baza de date.
- Template-uri: Generarea dinamică a HTML-ului.

==================================================
CONCLUZIE
==================================================

BookVoyager demonstrează eficiența Python și a Django Framework în crearea unei aplicații web sigure și scalabile. Caracteristici precum gestionarea automată a bazei de date (ORM) și procesarea logică server-side asigură o platformă robustă, imposibil de realizat doar prin tehnologii frontend.
